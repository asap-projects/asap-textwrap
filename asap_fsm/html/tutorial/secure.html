Tutorial: A secure door

Tutorial: A secure door¶

In this last step of the tutorial, we’ll make our door super smart. We’re going to add a lock code that can be set when locking the door and that will be required to unlock it later.

We’ll assume that the lock code is entered suing some kind of UI interacting with the user, and which provides it to us in the form of an integer.

The new state machine can be represented by the following diagram:

Besides an additional state, we’ll introduce two new events Lock and Unlock. Now, once the door is locked, the only way to open it, is to unlock it first.

  1#include <fsm/fsm.h>
  2
  3#include <iostream>
  4
  5using asap::fsm::ByDefault;
  6using asap::fsm::Continue;
  7using asap::fsm::DoNothing;
  8using asap::fsm::Maybe;
  9using asap::fsm::On;
 10using asap::fsm::StateMachine;
 11using asap::fsm::Status;
 12using asap::fsm::TransitionTo;
 13using asap::fsm::Will;
 14
 15struct OpenEvent {};
 16struct CloseEvent {};
 17
 18struct LockEvent {
 19  uint32_t newKey; // the lock code chosen by the user
 20};
 21
 22struct UnlockEvent {
 23  uint32_t key; // the lock key entered when unlocking
 24};
 25
 26struct ClosedState;
 27struct OpenState;
 28struct LockedState;
 29
 30struct ClosedState
 31    : Will<ByDefault<DoNothing>, On<LockEvent, TransitionTo<LockedState>>,
 32          On<OpenEvent, TransitionTo<OpenState>>> {
 33  using Will::Handle;
 34
 35  static auto OnEnter(const UnlockEvent & /*event*/) -> Status {
 36    std::cout << "   > door is closed - unlocked\n";
 37    return Continue{};
 38  }
 39
 40  template <typename Event> auto OnEnter(const Event & /*event*/) -> Status {
 41    std::cout << "   > door is closed\n";
 42    return Continue{};
 43  }
 44
 45  [[nodiscard]] static auto Handle(const CloseEvent & /*event*/) -> DoNothing {
 46    std::cerr << "Error: the door is already closed!\n";
 47    return DoNothing{};
 48  }
 49};
 50
 51struct OpenState
 52    : Will<ByDefault<DoNothing>, On<CloseEvent, TransitionTo<ClosedState>>> {
 53  using Will::Handle;
 54
 55  template <typename Event> auto OnEnter(const Event & /*event*/) -> Status {
 56    std::cout << "   > door is open\n";
 57    return Continue{};
 58  }
 59
 60  [[nodiscard]] static auto Handle(const OpenEvent & /*event*/) -> DoNothing {
 61    std::cerr << "Error: the door is already open!\n";
 62    return DoNothing{};
 63  }
 64};
 65
 66struct LockedState : ByDefault<DoNothing> {
 67  using ByDefault::Handle;
 68
 69  explicit LockedState(uint32_t key) : key_(key) {
 70  }
 71
 72  auto OnEnter(const LockEvent &event) -> Status {
 73    std::cout << "   > door is locked with new code(" << event.newKey << ")\n";
 74    key_ = event.newKey;
 75    return Continue{};
 76  }
 77
 78  [[nodiscard]] auto Handle(const UnlockEvent &event) const
 79      -> Maybe<TransitionTo<ClosedState>> {
 80    if (event.key == key_) {
 81      return TransitionTo<ClosedState>{};
 82    }
 83    std::cerr << "Error: wrong key (" << event.key
 84              << ") used to unlock door!\n";
 85    return DoNothing{};
 86  }
 87
 88private:
 89  uint32_t key_;
 90};
 91
 92using Door = StateMachine<ClosedState, LockedState, OpenState>;
 93
 94auto main() -> int {
 95  Door door{ClosedState{}, LockedState(0), OpenState{}};
 96
 97  constexpr int lock_code = 1234;
 98  constexpr int bad_code = 2;
 99
100  std::cout << "-- Starting\n";
101
102  std::cout << "-- sending open event\n";
103  door.Handle(OpenEvent{});
104
105  std::cout << "-- sending close event\n";
106  door.Handle(CloseEvent{});
107
108  std::cout << "-- sending lock event (" << lock_code << ")\n";
109  door.Handle(LockEvent{lock_code});
110
111  std::cout << "-- sending unlock event (" << bad_code << ")\n";
112  door.Handle(UnlockEvent{bad_code});
113
114  std::cout << "-- sending unlock event (" << lock_code << ")\n";
115  door.Handle(UnlockEvent{lock_code});
116}

If we run that example, this is what we get:

-- Starting
-- sending open event
  > door is open
-- sending close event
  > door is closed
-- sending lock event (1234)
  > door is locked with new code(1234)
-- sending unlock event (2)
Error: wrong key (2) used to unlock door!
-- sending unlock event (1234)
  > door is closed - unlocked

The new implementation demonstrates many advanced features offered by the asap::fsm library:

  • using templates SFINAE to define custom handlers for specific events,

  • passing data in events,

  • storing data in states

Additionally, the API allows for passing data during transitions from one state to another. Check the documentation of TransitionTo for additional details on that technique.
